Comment créer un site web (ou un blog)

June 10, 2018

Vous voulez créer un site web mais craignez que ce ne soit trop difficle, long, ou cher ?

Pourtant aujourd’hui, créer un site web professionnel ne requiert pas de compétences particulières ni de beaucoup de temps, si ce n’est trente minutes de concentration.

Quant au prix, il y en a pour tous les budgets. De 0€ à 4€/mois.

Dans cet article, je vais vous montrer trois aproches pour créer un site web:

  1. Créer un site professionnel avec WordPress
    Pour avoir un site robuste, simple et rapide à mettre en place, intuitif et personalisable sans jamais avoir à toucher une ligne de code, il faut créer son site avec WordPress.
    L’avantage de cette méthode est qu’elle est très rapide et simple à mettre en place.
  2. Créer un site ou un blog avec Github Pages
    L’avantage de cette méthode, c’est qu’elle est gratuite. Vous n’aurez à payer que le nom de domaine si vous en voulez un.
    En échange, il faudrat plus de temps et être familiarisé avec Git, Github et la syntaxe Mardown.
    Si vous désirez personnaliser le thème choisi, il faudra mettre la main au code.
  3. Créer un site avec Netlify
    Cette option est faite principalement pour les développeurs ou pour ceux qui peuvent embaucher un développeur. Elle à tous les avantages. D’un côté la gratuité et performance des sites statiques, d’un autre, la puissance d’un CMS.

Créer un site web est donc un compromis.
Choisissez entre ces trois approches en fonction de vos besoins, budget et domaine de compétences.

Plugins WordPress indispensables

Le plugins sont les pillier sur lequel wordpress tient.

Une des raisons pour lesquelles on utilise le CMS Wordpress plutot qu’un autre c’est qu’il a une communauté bien plus large. Ce qui veut dire plus de thèmes et plus de plugins.

Ces plugins sont des fonctionnalités pré-développées par des développeurs pour la pluspart très compétents et vous n’avez qu’a les installer gratuitement. C’est l’une des forces de wordpress.

Voilà quelques plugins indisepsables pour vous aider dans votre aventure.

Statistiques de visite
À moins que vous souhaitiez mettre les mains dans la boue pour installer Matomo (anciennement piwik) qui est une solution opensource utilisée nottament par le fournisseur d’accès à internet Free et le magazine Forbes, il va falloir installer Google Analytics. Après avoir créé un compte, il va falloir le configurer sur wordpress.
Les principaux plugins sont Analytify et GADWP. Les deux ont une version gratuite et affichent vos statistiques sur le tableau de bord.

SEO
Votre position sur la page de résultat de google vas dépendre de la qualité du SEO de votre site et articles. Il n’est pas toujours facile de mettre en place de manière régulière toutes les meilleurs pratiques et de rester à jour par rapport aux évolutions de google. Heureusement, le plugin Yoast SEO est venu résoudre ce problème. À l’aide de pastilles vertes, oranges ou rouges, ils vous indique ce qui est bon ou pas au niveau de l’article que vous êtes entrain de rédiger. De plus, il crée automatiquement les balises opengraph et twitter cards qui améliorent l’apparence de vos articles lorsqu’ils sont partagés sur facebook et twitter.

All In One Schema.org Rich Snippets est un autre plugin à explorer car il permet d’améliorer l’apparence de votre lien sur les résultats de recherche. Cela en ajoutant des Rich Snippets a vore site.

Améliorer dramatiquement les performances du site
WP Super Cache est un plugin développé par Automattic, l’équipe qui se cache derrière WordPress. Ce plugin est plus simple à configurer que ses concurrents et va vous permettre de booster les performances de votre site. N’oubliez pas de l’installer avant d’avoir beacoup de visites, votre serveur vous en remerciera. Si vous avez besoin d’aide pour le configurer, lisez ce guide

Proposer un meilleur contenu
Il serait dommage que vos visiteurs partent trop tôt parceque vous avez failli a votre mission qui est de leur proposer plus de contenu.
Deux plugins sont là pour vous sauver la mise. Premièrement Yet Another Related Posts Plugin vas vous permettre de proposer des articles similaires à celui qui est entrain d’être lu grâce a un algorithme qui analyse le contenu de chaque article, ainsi que son titre, ses catégories et ses tags. Finalement, le plugin WordPress Popular Posts permet de créer une liste d’articles triés par popularité. Les articles populaire son rarement populaires par hazard, veillez a bien les mettre en valeur. Dans la sidebar ou le footer par exemple.

Créer un site avec Github Pages

  1. Vous avez besoin d’un compte Github. Connectez-vous ou inscrivez-vous.
  2. Créez un répertoire GitHub avec le nom votrepseudo.github.io. Où votrepseudo est le pseudo défini en étape 1. C’est important pour que cela marche. Ce sera l’url du site.
  3. Rendez-vous dans l’onglet “settings” ou vous trouverez les réglages
  4. Défilez la page jusqu’a la section GitHub pages
  5. Choisissez votre thème
  6. Personnalisez votre première page en markdown et cliquez sur “commit” en bas de page pour sauvegarder et publier les changements.

Votre site est en ligne ! Vous pouvez y accéder tapant l’url définie en étage 3. ( pseudo.github.io )

Configurez votre propre nom de domaine

Si vous avez acheté votre nom de domaine sur NameCheap, vous pouvez maintenat l’associer à votre site web.

Je recommande NameCheap car ils ont un rapport qualité-prix excellent, sont honnètes et offrent WhoisGuard gratuitement sur tout les noms de domaines pour protéger votre vie privée.

Une fois que vous avez votre nom de domaine, suivez les étapes.

  1. Redirigez les DNS Nameserver vers ceux de github.

    1. Depuis votre tableau de bord NameCheap, cliquez sur “Domain List” Puis sur “Manage” à coté du nom de domaine correspondant.
    2. Dans la section “NAMESERVERS”, choisissez “Custom DNS”.
    3. Ajoutez l’url actuelle de votre site. celle que vous avez définis en étape 2 (votrepseudo.github.io)
      Dans mon cas, se serais: scriptor.github.io

  2. Ajoutez un fichier CNAME à la source de votre site.
    Ajoutez un fichier vierge, appelé CNAME. Il ne contiendra qu’une seul ligne. Votre nom de domaine.
    Exemple: scriptor.io où google.com

  3. Activez le HTTPS

Fonctionnement d’un site statique
On dit qu’un site est statique lorsque seul sont hébergés les fichier HTML, CSS et JavaScript. Le site statique que l’on va créer a un serveur. C’est le générateur de sites statiques. La différence avec un site dynamique c’est qu’il ne tourne pas tout le temps. Un site dynamique génère les pages sur demande tandis que le générateur génère toutes les pages d’un coup, puis nous laisse héberger les fichiers fichiers produits qui sont statiques.

Bénéfices et inconvénients d’un site statique
Performances extremes, plus rapide que 98% des sites web testés avec Pingdom. Complètement chargé en 376 millisecondes.

Il n’y a pas de bases de données à mettre en place et pas de serveurs à configurer. Dans beaucoup de cas, vous n’avez même pas besoin de savoir épeler HTML.

Réserver et Acheter un Nom de Domaine

La création d’un site statique est gratuit, ainsi que l’hébergement.

En revanche, si vous souhaitez avoir votre propre nom de domaine du style scritptor.io plutôt qu’un sous domaine qui ressemblerait, par exemple à scriptor.github.io dans le cas de la création d’un site web sur GitHub pages ou scriptor.netlify.io dans les cas de netlify.

Parmi les fournisseur de noms de domaines, je recommande NameCheap car il est économique, rapide et respectueux de la vie privée.
J’ai récemment migré tous mes noms de domaines de OVH à NameCheap.
De plus, contrairement a beaucoup de ses concurrents, ils sont honnêtes.
Le prix affiché est le vrai prix. Celui qui sera affiché sur la facture. Pas un prix hors taxes ou soumis à conditions.
Ils intègrent WhoisGuard gratuit à vie sur tous leurs noms de domaine.

WhoisGuard est un service qui protège votre vie privée.
Le Whois est une base de donnée publique ou sont enregistrées les coordonnées de tous les détenteurs de noms de domaines.

Quand vous achetez un nom de domaine, vous devez donner vos coordonnées complètes.
Nom, Prénom, Adresse Postale, Email, Numéro de Téléphone etc..

C’est obligatoire.

On ne peut pas réserver un nom de doamine avec de fausses informations. Sinon il n’est pas vraiment à nous et ils peuvent le reprendre à tout moment.

Ces données sont ensuite associé a votre nom de domaine et n’importe qui peut les consulter.

C’est un problème…

Si quelqu’un de malveillant atterri sur votre blogue personnel par exemple, vous ne voulez pas qu’il ait accès à ces données.
Avec WhoisGuard activé, le whois est rempli avec leurs données. Rien sur vous, pas même un indice.

Ils offrent donc une protection raisonnable. Meilleur que le service OWO de OVH.

Choix du générateur de sites statiques

Faut-il choisir Jekyll, Gatsby, Hexo ou Hugo ? Ou un autre ?

Les principales différences entre ses options le language de programmation utilisé et la taille de la communauté.

Ce dernier facteur est important.

La particularité d’un projet open-source c’est qu’il repose sur une communauté pour signaler et corriger les bugs. Développer des plugins, une documentation, aider les nouveaux a se familiariser avec l’écosystème et participer au développement du projet.

Plus la communauté d’un projet est grande, plus il a d’avenir.

Le language de programmation importe si vous êtes développeur.
Ainsi, vous pouvez faire vos propres modifications, développer de nouvelles fonctionnalités et participer au développement de l’application.

Jekyll est le champion vieillissant. Le plus populaire. Le premier générateur de sites statiques. Mais il se fait vieux. Il est écrit en Ruby, a le problème d’avoir le build time le plus lent. Certaines personnes laissent tomber jekyll et se tournent vers les nouveaux GSS car leur site a trop grandi et jekyll a du mal a le compiler. Malgré tout, Jekyll est encore le N°1 au niveau de la taille de la communauté. Il a l’avantage non négligeable d’avoir la plus grosse communauté et l’avantage de l’ancienneté. À choisir si vous aimez le language Ruby ou si vous n’avez pas envi de vous casser la tête. ce cador a peu de problèmes et si vous rencontrez un problème, il y a de grande chance que quelqu’un d’autre l’ait eu avant vous et qu’il ait été résolu par la communauté. A éviter si vous prévoyez de créer un gros site. Le site compilé sera très rapide comme pour tout les autres GSS mais le temps de compilation peut vous envoyer à l’asile ! Potentiellement plus de thèmes, plus de soutien.

Gatsby est une option assez impressionnante. Mon choix pour ce blog. Il a la particularité d’être développé avec React (Comprendre Javascript + fonctionnalité additionnelles). Il bénéficie d’un écosystème avec la croissance la plus rapide et tout le soutien de la communauté React. Il est facile à utiliser, très bien documenté et intègre les meilleures pratiques de développement, vitesse et SEO. On a toujours dit que react n’était pas adapté a la création de blogues ou sites statiques. Gatsby bien de changer la donne.

Hexo est un générateur de sites statiques créé avec NodeJS (Javascript). Il a u Il peut se vanter d’avoir une des plus grosses communauté et un build time très rapide. Le défaut majeur de cet outil, pour une personne occidentale est que la plupart de la communauté est asiatique?.

Hugo est le petit nouveau. Sa particularité est qu’il est écrit en GO, un language de programmation récent, créé par google. Il n’offre pas de nouveautés majeurs par rapport a ses concurrents. C’est une alternative viable à Jekyll si vous voulez améliorer votre build time.
